# 项目最终目标

本项目的最终目标是建设一个面向实际盘点场景的“物资绕拍视频自动计数系统”。

系统希望让用户通过手机或相机围绕一堆物资拍摄一段视频，上传后由系统自动估算目标物资数量，并给出可追溯、可复核、可持续优化的计数依据。

## 系统定位

当前项目不只是一个针对单个样例视频的计数脚本，而是一个从工程 demo 逐步演进到实际盘点工具的原型系统。

它的核心定位是：

1. 自动估算物资数量。
2. 输出可视化证据和置信度。
3. 在算法不确定时支持人工复核。
4. 将人工修正结果回流为训练数据。
5. 持续提升模型和计数流程的可靠性。

## 目标使用场景

典型使用流程如下：

1. 用户使用手机或相机围绕目标物资缓慢拍摄一段视频。
2. 用户将视频上传到系统。
3. 系统自动检查视频质量。
4. 系统抽取关键帧。
5. 系统检测每帧中可见的目标物资。
6. 系统识别同一物资在不同视角中的重复出现。
7. 系统融合多帧、多视角结果，估算最终数量。
8. 系统输出数量、置信度、异常提示和可视化证据。
9. 对低置信度或有争议的结果，用户可以进行人工复核和修正。
10. 系统保存修正结果，用于后续模型训练和算法优化。

## 核心能力

最终系统应具备以下能力：

1. 视频上传与任务管理。
2. 视频质量检测，包括模糊、曝光、分辨率、运动速度和目标占比。
3. 关键帧抽取与筛选。
4. 目标检测或实例分割。
5. 多视角目标匹配与重复计数抑制。
6. 数量估算与置信度评估。
7. 可视化证据展示，包括关键帧标注图、检测明细和诊断图。
8. 人工复核，包括删除误检、补充漏检和确认最终数量。
9. 数据闭环，包括保存原始视频、关键帧、检测结果、人工修正和真实数量。
10. 后续扩展到其他类似物资类型。

## 核心价值

本系统的核心价值是降低人工盘点成本，提高盘点效率，并在视觉算法无法完全确定时，通过证据展示和人工复核保证结果可信。

它追求的不是在所有复杂遮挡场景中完全自动、完全精确地计数，而是建立一个实际可用的工作流：

```text
自动估算
  -> 可视化证据
  -> 置信度判断
  -> 人工复核
  -> 修正数据回流
  -> 模型持续优化
```
