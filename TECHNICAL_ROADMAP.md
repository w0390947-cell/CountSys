# 物资绕拍视频自动计数系统技术路线

## 1. 目标定义

本系统目标是基于一段围绕物资或物品拍摄的视频，自动估算目标物资数量，并输出可追溯的计数依据。

当前样例场景是：竹竿包裹的圆柱形零件成堆摆放，用户手持设备绕拍，系统需要估算主物资堆中大约有多少件圆柱形零件。

系统最终应具备以下能力：

1. 自动读取视频并抽取关键帧。
2. 自动检测每帧中可见的目标物资。
3. 对同一物资在多个视角中的重复出现进行去重。
4. 输出估算数量、置信度、可视化证据和异常提示。
5. 支持后续扩展到其他类似物资类型。

## 2. 当前模型框架状态

当前已经完成一个面向样例视频的深度学习模型框架：

- 输入：`b7763a0682d156294de373ad97e2c544.mp4`
- 抽帧入口：`python3 scripts/extract_frames_for_labeling.py`
- 训练入口：`python3 scripts/train_single_video_yolo.py`
- 模型计数入口：`python3 scripts/count_with_yolo_model.py`
- Web demo：`python3 server.py`

当前模型路线的技术流程：

1. 使用 OpenCV 从视频中按固定间隔抽取关键帧。
2. 人工标注关键帧中的圆柱端面，形成 YOLO 数据集。
3. 使用 YOLO 训练端面检测模型。
4. 使用训练好的模型对关键帧进行检测。
5. 统计每帧检测数量，并保存标注图和检测明细。
6. 使用多视角关键帧中的强视角结果进行鲁棒融合，得到最终估算数量。

当前框架的定位是工程原型，不是生产级系统。它用于验证深度学习训练、推理、计数和 Web 展示链路。

## 3. 总体技术架构

建议系统分为 7 个核心模块：

1. 视频采集与质量检测模块
2. 关键帧抽取模块
3. 目标检测与实例分割模块
4. 多视角相机运动估计模块
5. 跨帧目标匹配与去重模块
6. 计数融合与置信度评估模块
7. 可视化审查与人工修正模块

推荐主流程：

```text
视频输入
  -> 视频质量检测
  -> 关键帧抽取
  -> 目标检测/实例分割
  -> 相机位姿估计
  -> 跨帧目标关联
  -> 空间聚类去重
  -> 数量估算
  -> 结果可视化
  -> 人工复核/反馈
```

## 4. 第一阶段：把 demo 做稳

目标：在当前样例视频及同类拍摄视频上，把估算结果稳定在可接受范围内。

### 4.1 输入规范

先制定拍摄规范，降低算法难度：

1. 视频时长建议 10 到 30 秒。
2. 摄像头围绕目标物资缓慢移动一圈。
3. 保持目标物资占画面主体区域。
4. 避免快速晃动、过曝、强反光和严重遮挡。
5. 尽量保证每一侧端面都有清晰视角。
6. 背景中如果有同类物资，需要明确是否纳入统计。

### 4.2 视频质量检测

在正式计数前增加质量检测：

- 分辨率检测：过低分辨率直接提示重新拍摄。
- 模糊检测：使用 Laplacian 方差判断模糊帧。
- 运动过快检测：相邻帧特征点位移过大时报警。
- 曝光检测：过暗、过亮、局部过曝时报警。
- 目标占比检测：目标区域太小或离镜头太近时报警。

输出示例：

```json
{
  "quality_passed": true,
  "blur_ratio": 0.08,
  "over_exposure_ratio": 0.03,
  "usable_frame_count": 42
}
```

### 4.3 关键帧抽取优化

当前 demo 是固定间隔抽帧。后续建议改成质量驱动抽帧：

1. 固定间隔粗抽帧。
2. 删除模糊帧、过曝帧、重复帧。
3. 根据相机视角变化选择代表帧。
4. 保证前、后、左、右、斜侧等视角都有覆盖。

推荐策略：

- 使用 ORB/SIFT 特征变化估计视角差异。
- 使用图像清晰度分数筛选高质量帧。
- 使用聚类方法从视频中选出 20 到 60 张关键帧。

## 5. 第二阶段：目标检测模型升级

目标：训练专用检测模型，降低误检和漏检。

### 5.1 检测对象定义

当前物资是竹竿包裹的圆柱形零件。建议优先检测两个对象：

1. `part_end_cap`：圆柱零件端面。
2. `part_body`：被竹竿包裹的圆柱主体。

其中端面更适合计数，主体更适合辅助判断目标区域和姿态。

### 5.2 数据采集

至少采集以下数据：

- 20 到 50 个同类物资绕拍视频。
- 不同角度、不同距离、不同光照的视频。
- 包含背景同类物资、遮挡、堆叠、模糊等困难场景。
- 每个视频保留人工盘点真实数量，作为评估基准。

建议第一批数据规模：

- 视频：30 条。
- 抽帧图片：每条 30 到 80 张。
- 标注图片：约 1000 到 2000 张。

### 5.3 标注规范

推荐标注方式：

1. 对清晰可见的端面标注实例框或实例 mask。
2. 对明显遮挡但仍可判断的端面也标注。
3. 对不可见但理论存在的物资不要标注。
4. 标注背景同类物资时加区域标签，用于区分是否纳入统计。

标注类别建议：

```text
part_end_cap
part_body_visible
background_same_type
ignore_region
```

如果后续要支持人工复核，建议保留每个实例的唯一编号或至少保留端面中心点。

### 5.4 模型选择

第一版推荐：

- 检测模型：YOLOv8/YOLOv10/RT-DETR
- 实例分割模型：YOLO-seg 或 Mask R-CNN
- 辅助分割：SAM/SAM2

落地建议：

1. 先训练 `part_end_cap` 检测模型。
2. 如果端面边界复杂，再升级到实例分割。
3. 如果竹竿遮挡严重，引入 `part_body_visible` 辅助判断。

模型输出应包含：

```json
{
  "bbox": [x1, y1, x2, y2],
  "center": [cx, cy],
  "score": 0.93,
  "class": "part_end_cap",
  "mask": "optional"
}
```

## 6. 第三阶段：多视角去重

目标：避免同一物资在多个角度重复计数。

### 6.1 短时跟踪

在相邻帧或短间隔关键帧之间，使用跟踪算法关联同一目标：

- ByteTrack
- OC-SORT
- DeepSORT

适用范围：

- 相机移动较慢。
- 相邻帧目标位移连续。
- 遮挡不太严重。

输出：

```json
{
  "track_id": 12,
  "frame_range": [30, 96],
  "detections": [...]
}
```

### 6.2 多视角空间归一

对于绕拍视频，仅靠 2D 跟踪不够。需要估计相机运动：

可选路线：

1. 轻量路线：ORB/SIFT + 单应矩阵
2. 中等路线：OpenCV SfM 或局部 BA
3. 稳定路线：COLMAP
4. 实时路线：ORB-SLAM3
5. 深度路线：单目深度估计 + 相机位姿估计

生产系统推荐 COLMAP 或 SLAM 路线。

原因：

- 当前物资堆是三维结构，不是平面。
- 单应矩阵只能作为近似，容易把不同深度的目标错误合并。
- 真正去重需要相机位姿和目标三维位置。

### 6.3 检测点三维定位

如果使用 COLMAP/SfM：

1. 从关键帧中恢复相机位姿。
2. 对每个检测框中心或 mask 区域提取局部特征。
3. 将同一物资在多个视角中的观测三角化。
4. 得到目标的近似三维点或三维区域。
5. 对三维点进行聚类。

输出结构：

```json
{
  "object_id": 37,
  "observations": [
    {"frame": 12, "bbox": [100, 200, 140, 240]},
    {"frame": 18, "bbox": [120, 210, 160, 250]}
  ],
  "position_3d": [1.2, -0.4, 3.7],
  "confidence": 0.82
}
```

### 6.4 外观特征辅助匹配

由于同类物资外观很相似，不能只依赖 ReID。但 ReID 可以作为辅助：

- 端面颜色
- 端面大小
- 端面椭圆方向
- 周围竹竿纹理
- 所在层级和相对位置

推荐组合评分：

```text
match_score =
  位置一致性 * 0.45
  + 视角几何一致性 * 0.25
  + 尺寸一致性 * 0.15
  + 外观相似度 * 0.15
```

## 7. 第四阶段：计数融合与置信度

目标：系统不仅输出一个数字，还要说明这个数字是否可信。

### 7.1 多来源计数融合

建议融合以下结果：

1. 最佳视角单帧计数。
2. 多帧跟踪轨迹数。
3. 三维空间聚类数。
4. 密度估计补偿数。
5. 人工修正反馈数。

融合逻辑：

```text
如果三维重建质量高：
    以三维聚类数为主
否则如果最佳视角清晰：
    以最佳视角计数为主
否则：
    输出低置信度结果并提示人工复核
```

### 7.2 置信度设计

置信度可以由以下因素决定：

- 可用关键帧数量
- 平均检测置信度
- 多视角覆盖程度
- 相机位姿估计成功率
- 跨帧匹配稳定性
- 遮挡程度
- 不同计数方法之间的差异

输出示例：

```json
{
  "estimated_count": 63,
  "confidence": 0.72,
  "count_range": [55, 70],
  "review_required": true,
  "reason": "gravel false positives and partial occlusion detected"
}
```

### 7.3 异常提示

系统应主动提示：

- 视频过模糊。
- 视角覆盖不足。
- 目标被遮挡严重。
- 背景中存在同类物资。
- 计数结果波动过大。
- 检测模型置信度过低。

## 8. 第五阶段：人工复核闭环

目标：让系统从“自动估算”升级为“可审计的半自动盘点工具”。

建议界面能力：

1. 显示估算总数。
2. 显示关键帧检测框。
3. 显示每个疑似物资编号。
4. 支持用户删除误检。
5. 支持用户补充漏检。
6. 支持导出修正后的最终数量。
7. 将人工修正结果回流为训练数据。

这是生产系统非常关键的一步。视觉计数在遮挡场景下很难做到完全准确，人工复核可以显著提升可信度。

## 9. 评估体系

### 9.1 核心指标

建议用以下指标评估：

- 绝对误差：`abs(pred_count - true_count)`
- 相对误差：`abs(pred_count - true_count) / true_count`
- 平均绝对误差：MAE
- 平均相对误差：MAPE
- 检测召回率：Recall
- 检测精确率：Precision
- 重复计数率
- 漏计率
- 人工复核耗时

### 9.2 分场景评估

测试集应按难度分层：

1. 简单：光照好、端面清晰、遮挡少。
2. 中等：轻微遮挡、背景复杂。
3. 困难：强遮挡、同类背景、端面只露出部分。
4. 极难：模糊、过曝、视角覆盖不足。

不同阶段目标：

```text
demo 阶段：相对误差 20% 到 35% 可接受
试点阶段：相对误差 10% 到 20%
生产阶段：相对误差 5% 到 10%，困难场景允许人工复核
```

## 13. 主要风险与应对

### 13.1 遮挡导致漏计

风险：堆叠物资中很多零件不可见，任何视觉系统都无法完全准确推断。

应对：

- 强化拍摄规范。
- 多角度拍摄。
- 引入密度估计补偿。
- 输出置信区间而不是单一数字。
- 高风险场景进入人工复核。

### 13.2 背景同类物资干扰

风险：背景里有同类圆柱零件，容易被误计。

应对：

- 加入目标区域选择。
- 使用人工或自动 ROI。
- 训练 `background_same_type` 或 `ignore_region` 类别。
- UI 中允许用户圈定盘点区域。

### 13.3 重复计数

风险：同一个物资在绕拍视频中多次出现。

应对：

- 短时跟踪。
- 三维重建。
- 空间聚类。
- 多方法结果交叉验证。

### 13.4 标注数据不足导致泛化差

风险：如果只使用少量样例视频训练，模型容易过拟合，换光照、角度、背景或遮挡条件后效果下降。

应对：

- 按场景持续采集数据。
- 持续补充人工标注。
- 建立人工修正回流机制。

### 13.5 三维重建失败

风险：纹理少、画面模糊、运动过快时，SfM/SLAM 可能失败。

应对：

- 视频质量前置检测。
- 引导用户重新拍摄。
- 使用最佳视角计数作为降级策略。
- 使用标尺、转台或固定拍摄轨迹提升稳定性。

## 14. 推荐的下一步

建议立即推进以下事项：

1. 收集 5 到 10 条同类物资绕拍视频。
2. 为每条视频记录真实人工盘点数量。
3. 用当前 demo 批量跑这些视频，得到误差分布。
4. 选取误差最大的样例，分析误检、漏检、重复计数原因。
5. 制定第一版标注规范。
6. 标注 300 到 500 张关键帧，训练第一个端面检测模型。
7. 将 Web demo 的上传计数流程稳定接入模型推理接口。

短期最重要的不是继续堆复杂算法，而是尽快建立“数据、真实数量、误差分析、模型训练、人工修正”的闭环。
